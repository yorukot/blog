> 以下照我的面試順序排列

# 國立成功大學

嚴格來說這不太算面試，是上機考。對於專案仔非常之友善，整體方向確實偏專案沒錯，但因為有 copilot，所以大家基本上就是詠唱師，不斷請 AI 精進而已（？

更詳細內容我很推薦 [Wolf 的特選心得](https://wolf-yuan.dev/zh-tw/blog/srecuit/#%E6%88%90%E5%8A%9F%E5%A4%A7%E5%AD%B8%E4%B8%8A%E6%A9%9F)，寫得非常之詳細；我的記憶力實在不足以讓我把整套流程回想完整（

# 國立臺南大學

這是我面試的第一間大學，可以說也是最緊張的一間。

印象中是兩位教授，主要由其中一位負責發問。整體過程蠻輕鬆的，問的重點主要圍繞在我的開源專案 superfile，比較沒有往技術細節鑽。

# 國立高雄大學

這間我給的評價非常之低，整體感受就是：教授完全沒有用心。

面試形式是團體面試，一輪三位。第一題問「你來自哪裡、讀什麼學校、什麼科系」。我聽到的當下就滿臉問號，但我總不能中途離開，所以就繼續待著了。

後面有問到「大學想做什麼專題」。我回答了類似 dokploy、coolify 的專題方向，但教授似乎沒聽懂；可能也因為這樣，我拿到了一個偏低的成績。

整體而言體驗蠻糟糕的。光是為什麼第一題要問這種毫無緊要的問題就很問號了；再加上團體面試真的是個非常糟糕的做法，它很容易讓每個人都處在不公平的狀態。

# 宜蘭大學

整體面試氛圍很好，還有問到我備審最後一頁的 AI 相關內容，看得出來他們有在認真讀備審。最後甚至跟我說希望我能來宜蘭大學就讀。

### 小插曲

因為我從臺北搭客運到宜蘭，沒想到會遇到塞車，結果就這樣 delay 了。還好我有打給負責人，他說沒問題，幫我往後排就好。

但在現場排隊的過程中，他們似乎把我忘記了，還好我有去問，不然連海大也不用面了（

# 國立臺灣海洋大學

海洋大學是我覺得面最好的一間。

印象中兩位教授：一位偏專案，一位偏演算法。問專案的時候，superfile 的細節被問得很細，甚至有問"為什麼要用 Go"。我回答是因為要併發處理，他們就接著追問"那有沒有遇到 race condition"之類的問題，可以說是我所有大學裡面，被挖最深的一場。

不過在演算法教授那邊，我把 stack 跟 heap 的定義講反了。可能也因為這樣沒正取（？

### 小插曲

因為前面宜蘭大學大 delay，導致我必須從臺北市府轉運站坐 Uber 到海洋大學，直接噴掉 800（。
不過好在最後有趕上。

# 國立清華大學

這算是我最恐怖的一段面試記憶。

流程印象中是兩關：  
第一關用簡報自我介紹 10 分鐘；第二關抽演算法題，然後請你當場講解題思路。

## 第一關

自我介紹很順利講完後，教授開始詢問問題。細節我不是很記得，但基本上主要是針對 superfile 去問。印象中教授給了非常高的評價，當下我甚至以為我這次要穩了。

但結果告訴我肯定是第二關出了什麼大問題。

## 第二關

第二關要抽一題演算法。我抽到之後先想了 3 分鐘才開始講初步想法，接著就跟教授乾瞪眼大概 3 分鐘左右。

我每提出一個想法就馬上被打槍，~~真的好受傷~~。到最後我還是沒能給出一個像樣的解，甚至還掏出了一個 4^n 的解法，不愧是我專案仔。

## 後記

~~我嚴重且合理的懷疑清大只是想來看看我到底是誰而已~~

# 國立陽明交通大學

陽明交通大學的面試也非常舒服。

一進去先一分鐘自我介紹，講完就開始問問題。問題主要專注在 superfile 的社群管理跟大方向，比較沒有鑽技術細節。

唯一比較偏技術的提問是：用 Go 開發有沒有遇到什麼問題。
我回答：語言本身沒有，主要遇到的是 terminal image protocol 的適配問題。

值得一提的是，交大是唯一一個現場把 GitHub 打開在看我 GitHub 的學校，甚至還問我 PR 的狀況、我目前主要的職位在幹嘛之類的，可以說是非常認真了。
